CSS Keyframes Animations

What is a Keyframes Animations?

A list of what should happen over the course of the animation. You can specify which properties should change, how, and when.

There are 2 basic building blocks to keyframe animations:

  1. Keyframes - define the stages and styles of the animation.
  2. Animation Properties - assign @keyfromes to a specific CSS element and define how it is animated.

#1 Keyframes

Each @keyframes is composed of:

  • Name of the animation: A name that describes the animation. e.i. pulsate
  • States of the animation: Each stage of the animation is represented as a percentage. From (initial stage) 0% to (final stage) 100%. Multiple intermediate states can be added in between.
  • CSS Properties

CSS Syntax (vendor prefix omitted for this demo)

  • @keyframes (name-animation) {
  • (step 1 e.i. 0%) {
  • property: value;
  • }
  • (step 2 e.i. 100%) {
  • property: value;
  • }
  • }
  • @keyframes pulsate {
  • 0% {
  • transform: scale(0.1);
  • opacity: 0;
  • }
  • 60% {
  • transform: scale(1.2);
  • opacity: 0.6;
  • }
  • 100% {
  • transform: scale(1);
  • opacity: 1;
  • }
  • }

#2 Animation Properties

Animation properties do 2 things:

  1. Assign the @keyframes to the elements that you want to animate.
  2. Define how it is animated.

    3. You must add first two animation properties for the animation to take effect.

    • animation-name (required)
    • animation-duration (required)
    • animation-timing-function
    • animation-delay
    • animation-iteration-count
    • animation-direction
    • animation-fill-mode
    • animation-play-state

CSS Syntax - animation shorthand

  • .target {
  • color: purple;
  • animation: pulsate 3s infinite ease-out;
  • }

Additional Animation Properties

You can futher customize and create complex animations using the additional properties.

  1. Animation-timing-function: defines the speed curve or pace of the animation.

    As we went over in CSS Transform module, you can specify the timing with predefined timing options: ease, linear, ease-in, ease out, ease-in-out, and cubic-bezier curve for more advanced timing functions.

    To revivist timing function
  2. Animation-delay: specifies when the animation should start. This lets the animation sequence begin some time after it's applied to an element.

    The value is defined in seconds(s) or milliseconds(mil).

    loading

  3. Animation-iteration-count: specifies the number of times that the animation will play.

    Possible values: (Note: default is 1)

    • initial(defaults)

      Sets the iteration count to the default value of 1.

    • infinite

      Repeats forever.

    • inherit

      Inherits the value from the parent.

  4. Animation-direction: property specifies whether the animation should play forward, reverse, or in alternate cycles.

    Possible values:

    • normal(defaults)

      The animation plays forward each cycle. Each time animation resets to the beginning state (0%) and plays forward again (to 100%).

    • reverse

      The animation plays backwards. On each cycle the animation resets to the end state (100%) and plays backwards (to 0%).

    • alternate

      The animation reverses direction every cycle. On each odd cycle, the animation plays forward (0% to 100%). On each even cycle, the animation plays backwards (100% to 0%).

    • alternate-reverse

      The animation reverses direction every cycle.On each odd cycle, the animation plays in reverse (100% to 0%). On each even cycle, the animation plays forward (0% or 100%).

  5. Animation-fill-mode: specifies if the animation styles are visible before or after the animation executes.

    Possible values: (Note: Several values may be given, separated by commas.)

    • forwards

      After the animation is finished, the styles defined in the final keyframe (100%) are retained by the element.

    • backwards

      Before the animation (during the animation delay), the styles of the initial keyframe (0%) are applied to the element.

    • both

      The animation will follow the rules for both forwards and backwards, extending the animation properties before and after the animation.

    • none(default)

      The animation does not apply any styles to the element, before or after the animation.

    This grows and goes back
    .example-fill-mode:hover .fill.none { animation: grow 3s; }
    This grows and stays
    .example-fill-mode:hover .fill.forwards { animation: grow 3s; animation-fill-mode: forwards; }
  6. Animation-play-state: specifies whether the animation is playing or paused. If you are trying to have an element freeze in the last shown frame until we hover over it again, we can apply the animation from teh beginning, with animation-play-state: paused; and just change it on :hover to animation-play-state: running;
    Hover over and watch me spin!
    .example-play-state { animation-play-state: paused;}
    .example-play-state:hover { animation-play-state: running;}
